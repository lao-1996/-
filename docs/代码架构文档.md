## 项目概述

### 技术栈
- **核心语言**: JavaScript (ES6+)
- **图形渲染**: HTML5 Canvas 2D Context
- **音频处理**: Web Audio API
- **UI框架**: 原生HTML/CSS
- **构建工具**: 无需构建，纯前端应用

### 设计理念
- **模块化**: 功能分离，易于维护
- **性能优先**: 60FPS渲染，优化资源使用
- **用户友好**: 直观操作，清晰反馈
- **可扩展性**: 易于添加新功能

    if (screenShake > 0) {

#### 2.5 物理引擎
```javascript
// 物理更新循环
function updatePhysics() {
    // 1. 应用重力
    player.velocityY += CONFIG.GRAVITY;
    
    // 2. 更新位置
    player.x += player.velocityX;
    player.y += player.velocityY;
    
    // 3. 边界检测
    if (player.y > groundLevel) {
        player.y = groundLevel;
        player.velocityY = 0;
        player.isJumping = false;
    }
}

// 碰撞检测
function checkCollisions() {
    // 矩形碰撞检测算法
    function rectCollision(rect1, rect2) {
        return rect1.x < rect2.x + rect2.width &&
               rect1.x + rect1.width > rect2.x &&
               rect1.y < rect2.y + rect2.height &&
               rect1.y + rect1.height > rect2.y;
    }
}
```

## 数据流分析

### 游戏循环数据流
```
用户输入 → 游戏状态更新 → 物理计算 → 碰撞检测 → 
状态变更 → 音频/粒子/抖动效果 → 渲染 → 显示
```

### 状态转换
```
START → (点击开始) → PLAYING
PLAYING → (碰撞/掉落) → GAME_OVER
PLAYING → (过关对话) → DIALOG → PLAYING
PLAYING → (抵达终点) → VICTORY
GAME_OVER → (重新开始) → START
```

### 资源管理
- **内存管理**：粒子系统自动回收死亡粒子
- **音频资源**：动态合成，无需预加载
- **渲染优化**：只渲染屏幕内对象
- **事件监听**：正确的事件绑定和解绑

## 性能优化

### 渲染优化
1. **视口裁剪**：只渲染相机范围内的对象
2. **粒子池**：重用粒子对象，减少GC压力
3. **Canvas状态管理**：最小化save/restore调用
4. **分层渲染**：按渲染频率分离图层

### 内存优化
1. **对象池**：重复使用游戏对象
2. **及时清理**：移除不再使用的对象
3. **事件监听器管理**：避免内存泄漏

### 性能监控
```javascript
// 简单的FPS监控
let lastTime = 0;
let frameCount = 0;
let fps = 0;

function updateFPS(currentTime) {
    frameCount++;
    if (currentTime - lastTime >= 1000) {
        fps = frameCount;
        frameCount = 0;
        lastTime = currentTime;
        // 可以显示在调试界面
    }
}
```
